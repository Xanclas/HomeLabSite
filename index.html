HomeLAB BOM

Purpose and context:
This page has the purpose to share my Home LAB BOM. vSphere is mainly used for the infrastructure management and hosting, but the environment has also other network components that allow better scalability using Proxmox.
This home lab was built to test applications, develop automation, test infrastructure components, design/implement network topologies. Besides vSphere it contains other software for backups and Windows and Linux machines.

The following image shows a generic architecture of this setup, regarding how network was designed to be more scalable:



Brief description:
The ESXi on the yellow/brown square represents the host where the NSX-T Edge VMs are running. This includes the T0 and T1, a with configured eBGP.
The Proxmox on the blue square, hosts a Pfsense and a VyOS router, which this last creates the eBGP session with the T0 Edge VM. There’s another VyOS router that runs on ESXi and connects directly to the Management network of the vCenter and ESXis.
Dedicated subnets/VLANs are used to connect the VyOS on Proxmox to the T0 Edge VM (VLAN: 40), as the Pfsense to both VyOs routers (VLAN: 20).
Route distribution is set to enable required NSX-T workloads (like Veeam for example) to connect the Management network of the vCenter. This management network also hosts a DNS server, which is used by all the workloads.
Internet access is provided by the Pfsense, by S-NAT configuration.
